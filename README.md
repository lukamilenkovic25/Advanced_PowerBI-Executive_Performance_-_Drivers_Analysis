## Data Model & Assumptions
The project uses a star schema design:
- Fact_Sales – transaction‑level sales data
- Dim_Date – calendar and time intelligence
- Dim_Product – product hierarchy
- Dim_Customer – customer and segment attributes
- Dim_Region – geographical regions
Relationships are one‑to‑many and single‑direction to ensure clean filter behavior and scalable DAX logic.
